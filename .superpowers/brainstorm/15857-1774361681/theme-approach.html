Theme Approach

The site already has light/dark mode toggle. How should the editor integrate?

A

Inherit Site Theme

Editor follows the site's light/dark toggle using var(--md-*) CSS variables. Light mode = light editor, dark mode = dark editor. Seamless integration, no extra controls. The mockups I've shown are dark — but light mode would look equally polished using the site's existing palette.

B

Always Dark (Editor Only)

Editor always uses a dark theme regardless of site mode — like a code editor embedded in a page. Sets its own CSS variables. Consistent "pro editor" vibe, but might feel jarring if the rest of the site is in light mode.

C

Enhanced Inherit

Inherits light/dark from the site toggle, but adds its own accent palette (purple/indigo) on top for active states, toolbar highlights, and status indicators. Best of both worlds: fits the site's scheme while having a distinct "editor" identity.
